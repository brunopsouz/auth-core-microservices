Jornada principal do browser

Cadastro

Frontend envia nome, e-mail, contato e senha. O usuário nasce pendente de verificação.

POST /api/auth/register

Verificação

Usuário informa o código OTP recebido por e-mail. A conta fica apta para login.

POST /api/auth/verify-email

Login

API valida credenciais, aplica rate limit, cria sessão e emite access JWT curto.

POST /api/auth/session/login

Cookies

O browser recebe sessão, access token e CSRF. O JWT não precisa ser lido por JavaScript.

Set-Cookie

Request comum

Endpoints de sessão validam a sessão ativa. APIs comuns/Gateway podem validar o JWT curto quando aplicável.

GET /api/auth/session/me

Refresh

Quando o access token expira, o backend valida a sessão server-side e emite novo JWT.

POST /api/auth/session/refresh

Sessões

Usuário lista dispositivos ativos e pode revogar uma sessão específica.

DELETE /api/auth/session/sessions/{sid}

Logout

AuthCore revoga a sessão principal e expira os cookies de autenticação e CSRF.

POST /api/auth/session/logout

Browser/PWA

O frontend opera com credenciais incluídas e header CSRF nas mutações autenticadas por cookie.

Formulário e-mail e senha
credentials: include cookies enviados automaticamente
X-CSRF-TOKEN proteção em refresh, logout e revogação

AuthCore API

Controller fino chama os casos de uso e cuida apenas de adaptação HTTP, cookies e resposta JSON.

LoginSessionUseCase cria sessão e token curto
RefreshBrowserSessionUseCase renova o access token
Logout/Revoke encerra sessão atual, todas ou uma sessão alvo

Infra + Domínio

A sessão é a fonte de controle. Banco mantém durabilidade; Redis apoia dados efêmeros e rate limit.

User, Password, Session invariantes de negócio
PostgreSQL sessão, refresh token, auditoria
Redis rate limit e cache/temporários opcionais

Fluxo browser recomendado

O access token curto fica em cookie HttpOnly. Revogar a sessão impede novas renovações; tokens já emitidos continuam válidos só até expirar. Para operações sensíveis, valide JWT e sessão ativa.

HttpOnly Secure SameSite=Lax CSRF obrigatório Rate limit no login

Fluxo alternativo por token

Para APIs, mobile e integrações, o AuthCore também expõe login/refresh/logout token-based. Nesse modo, access token e refresh token voltam no corpo da resposta.

POST /api/auth/token/login POST /api/auth/token/refresh POST /api/auth/token/logout
